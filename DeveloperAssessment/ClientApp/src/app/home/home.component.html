Welcome to my example SPA, built with:

  • ASP.NET Core and C# for cross-platform server-side code
  • Angular and TypeScript for client-side code
  • Bootstrap for layout and styling
  • Angular Material Material Design components for Angular

This leverages .net 6 as well as the following:

  • "Autofac" Version="6.4.0"
  • "Autofac.Extensions.DependencyInjection" Version="8.0.0"
  • "Microsoft.AspNetCore.SpaProxy" Version="6.0.6"
  • "Microsoft.Data.Sqlite.Core" Version="6.0.9"
  • "Microsoft.EntityFrameworkCore" Version="6.0.9"
  • "Microsoft.EntityFrameworkCore.Design" Version="6.0.9">
  • "Microsoft.EntityFrameworkCore.Sqlite" Version="6.0.9"
  • "Microsoft.EntityFrameworkCore.SqlServer" Version="6.0.9"
  • "Newtonsoft.Json" Version="13.0.1"
  • "System.Linq.Dynamic.Core" Version="1.2.20"

I used the standard dotnet tool commands to create the sqlite database as follows:

  • "dotnet ef migrations add InitialCreate"Creating and initializing the migrations
  • "dotnet ef database update"Creating the database, tables, and applying migrations
  • "dotnet ef database drop"(if necessary for dubugging) dropping the database
  • "dotnet ef migrations remove"(if necessary for debugging) removing migrations

Overall I liked the experience of running the exercise. If I were to expand it, I would probably break my bar-chart componet into two separate componets. One pertaining to the data grid and one pertaining to the chart then added together into the mat-drawer. I did not get around to enabling sorting but might be nice. I would probably add some additional endpoints for flexibility. Then I would start looking at how best to make the code more extensible. In this case I would probably start with refactoring more for solid and add unit testing then an IRequester interface (via Autofac just my usual DI tool) to make REST api calls to endpoints in the case that we wanted to expand on our design and make dlls for more apis we might be interested in. If this started to expand further past a a couple records might start looking at a higher grade data store (IE sql server) I already started moving in this direction adding SQLServer EF and Autofac from nuget and I will probably continue missing with the project after this interview for fun.
